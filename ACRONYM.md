# G.E.O.F.F. - Git-backed Evidence Operations Forensic Framework

## The Name

**G.E.O.F.F.** stands for **Git-backed Evidence Operations Forensic Framework**

### Breaking It Down

| Letter | Word | Meaning |
|--------|------|---------|
| **G** | **Git-backed** | Every action validated and committed to git for full reproducibility. Another investigator can clone the repo and reproduce every step. |
| **E** | **Evidence** | Core focus on digital evidence - disk images, memory dumps, network captures, logs, mobile devices. |
| **O** | **Operations** | 100% coverage of forensic operations via 9 specialist modules (32 functions). |
| **F** | **Forensic** | Built for DFIR - Digital Forensics and Incident Response. |
| **F** | **Framework** | Complete system: playbooks, validation, training, extensible architecture. |

## Why This Name?

1. **Git-backed** → The Critic validation pipeline commits every validation to git
2. **Evidence Operations** → SleuthKit, Volatility, YARA, Registry, Timeline, Network, Logs, Mobile
3. **Forensic Framework** → 18 PB-SIFT playbooks, CTF training data, conversational interface
