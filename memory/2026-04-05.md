## Lessons Learned

### What Went Wrong: Missed File Exfiltration

**The Mistake:** I saw the `Secret/Szechuan Sauce.txt` string in memory and reported "no evidence of theft" because I didn't see file *transfer* operations in the PCAP or disk artifacts. I was looking for the wrong evidence.

**What I Should Have Done:**
1. **Memory strings = file contents** — When you see `Secret/SECRET_beth.txt` followed by `"Space Beth is the real Beth"`, that's not a filename. That's the *actual file content* loaded into memory.
2. **Look for context** — The strings `NoJerry.txt` followed by `"You are a disappointment!"` proves the attacker *opened and read* the file, not just listed it.
3. **Understand Meterpreter behavior** — The malware was a Meterpreter payload. Meterpreter doesn't "copy files" in the traditional sense — it loads file contents into memory for exfiltration over the encrypted C2 channel.

**The Real Evidence:**
- ❌ File transfer in PCAP — NOT the right indicator
- ✅ File *contents* in memory strings — THE SMOKING GUN
- ✅ Active C2 session during timestamp — Means of exfiltration

**Rule for Future:** When investigating file exfiltration:
1. Search memory for *both* filenames AND file contents
2. Look for narrative text (sentences, quotes) — this proves reading, not just listing
3. Correlate with active C2 sessions — memory contents + C2 = exfiltration
4. Don't wait for PCAP file transfer evidence with Meterpreter/similar tools
